> **完成日期：** 2026-03-19
> **阶段状态：** ✅ 完成

---

## 项目概述

**产品名称：** 百家姓 AI 姓氏溯源助手

**一句话定位：** 中国首个「家族根目录」——用 AI 对话让每个人都能轻松找到自己的姓氏起源、家族脉络和文化传承。

**一句话人群定位：** 面向给孩子起名的年轻父母，解决「我从哪里来」的情感刚需。

## 关键决策汇总

### 战略

| 决策项 | 结论 |
|-------|------|
| **第一不公平优势** | 带权威正史出处的结构化姓氏根脉数据库 |
| **品类定位** | AI姓氏寻根助手（对外）/ 家族文化服务产品（对内） |
| **滩头阵地** | 给孩子起名的年轻父母 |
| **核心差异化** | 唯一把"AI对话+姓氏溯源+家族脉络"结合的产品 |

### 商业模式

| 决策项 | 结论 |
|-------|------|
| **定价带** | 99-399元（不做9.9元低价战） |
| **变现原则** | 基础溯源免费 + 增值服务分层付费 |
| **产品路线** | 小程序 → 抖音/小红书 → 独立APP |

### 内容策略

| 决策项 | 结论 |
|-------|------|
| **内容库** | S级8个 + A级12个 = 20个姓氏 |
| **模块结构** | 起源、郡望、堂号、家训、名人、起名思路 |
| **生产流程** | AI生成 + 兼职学生校对 |
| **SEO策略** | 公众号引流 + 小程序承载 |

### 视觉方向

| 决策项 | 结论 |
|-------|------|
| **视觉定位** | 新中式极简 + 温和专业 + 轻国风传承 + 干净AI对话 |
| **色彩系统** | 暖白背景 + 墨蓝强调 + 深红点缀（≤10%） |
| **排版风格** | 大量留白、书卷气、克制 |
| **AI对话** | 清爽、现代、流式输出 |

### 技术栈

| 决策项 | 结论 |
|-------|------|
| **前端** | 原生微信小程序 |
| **后端** | NestJS + MySQL + Redis |
| **AI** | Kimi/Moonshot API（流式输出） |
| **部署** | Linux云服务器（Ubuntu 24.04, 8GB） |

### 约束条件

| 约束类型 | 关键点 |
|---------|--------|
| **时间** | 60天MVP，3个月窗口期 |
| **团队** | 5-8人，5-15万预算 |
| **合规** | 零迷信、零夸大、内容可溯源 |
| **账号** | 个人微信账号（支付有限额） |

## 用户画像（核心）

### 核心用户：林晓（28岁）

- 一胎孕妈，新一线城市，月收入2w-3w
- 决策周期：孕4月→出生后7天
- 核心痛点：多方需求无法兼顾、只能拿名字拿不到根、全是套路
- 付费动机：解决家庭冲突、给孩子一个有根的名字

## 品牌语气

> **靠谱 = 坦诚 + 严谨 + 温暖**

- 年龄感：28-35岁，高学历家族成员
- 关系：专业顾问，权威但不傲慢
- 不确定时：直接说清争议，给出主流观点
- 红线：不说"大师"、"绝密"、"最准"

## 下一步行动

### 立即执行（0-30天）

1. **内容建设**：开始20个姓氏的内容整理（优先S级8个）
2. **技术开发**：小程序框架搭建 + 后端架构
3. **AI接入**：Kimi API接入 + 内容审核层

### 短期（30-60天）

1. MVP功能开发完成
2. 20个姓氏内容填充
3. 内部测试 + 微信审核提交

### 中期（60-90天）

1. MVP上线
2. 30天数据验证
3. 决定迭代方向
