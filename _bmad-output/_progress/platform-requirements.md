# Platform Requirements

## 技术栈决策

### 前端

| 组件 | 选择 | 说明 |
|-----|------|------|
| **平台** | 微信小程序（原生） | 首发唯一平台 |
| **框架** | 原生小程序开发 | 不使用Taro/Uni-app，保持轻量 |

### 后端

| 组件 | 选择 | 说明 |
|-----|------|------|
| **框架** | NestJS | TypeScript、企业级架构 |
| **数据库** | MySQL | 结构化数据存储 |
| **缓存** | Redis | 会话管理、热点数据 |
| **部署** | Linux云服务器（Ubuntu 24.04） | 已有 |

### AI

| 组件 | 选择 | 说明 |
|-----|------|------|
| **模型** | Kimi/Moonshot API | 中文理解强、API稳定 |
| **调用方式** | 流式输出（SSE） | 支持打字机效果 |
| **内容管控** | 独立审核层 + Kimi内置 | 固定回答边界 |

### 服务器配置

| 配置 | 规格 |
|-----|------|
| **系统** | Ubuntu 24.04 64位 UEFI |
| **CPU** | 4核 |
| **内存** | 8GB |
| **带宽** | 5Mbps |

## 技术约束

### 性能要求

- 首屏加载：< 3秒
- AI响应：流式输出，首字符 < 1秒
- API响应：< 500ms

### 安全要求

- AI内容必须经过审核层
- 禁止封建迷信表述
- 用户敏感数据（出生时辰）仅临时使用，不长期存储

### 合规要求

- 微信小程序审核要求
- 内容合规（无迷信、无政治、无宗教）
