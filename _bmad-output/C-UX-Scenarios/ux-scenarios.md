# 阶段 3: UX Scenarios - 用户场景设计

> **项目**：百家姓 AI 姓氏溯源助手  
> **阶段**：3 - UX Scenarios  
> **产出**：UX Scenarios

---

## 1. 核心用户场景

### 场景 1：首次体验 - 姓氏溯源

| 要素 | 内容 |
|-----|------|
| **场景名称** | 我的姓氏有什么故事？ |
| **用户** | 28岁孕妈林晓，首次使用 |
| **动机** | 好奇，想知道自己的姓氏起源 |
| **触点** | 小红书种草 → 小程序入口 |
| **流程** | 输入姓氏 → AI 智能匹配 → 展示姓氏起源 → 引导付费解锁完整报告 |

**关键交互：**
1. 极简首页，只有姓氏输入框 + "开始溯源"按钮
2. 姓氏输入后，1秒内展示姓氏起源卡片（免费预览）
3. "解锁完整家族故事"引导付费

---

### 场景 2：付费转化 - 完整报告

| 要素 | 内容 |
|-----|------|
| **场景名称** | 深入了解我的家族历史 |
| **用户** | 已被免费预览吸引，有付费意愿 |
| **动机** | 获得完整的姓氏文化、家族迁徙、名人故事 |
| **触点** | 免费预览 → 付费解锁 |
| **流程** | 查看预览 → 支付9.9元 → 获取完整报告 → 生成起名灵感 |

**关键交互：**
1. 报告预览页展示"未解锁"标识，激发好奇
2. 付费按钮突出，价格锚定（原价99，现价9.9）
3. 支付成功后，动画过渡到完整报告

---

### 场景 3：社交分享 - 获取社交货币

| 要素 | 内容 |
|-----|------|
| **场景名称** | 分享我的家族故事 |
| **用户** | 获得有价值内容，想要炫耀/分享 |
| **动机** | 获得社交认同，证明自己的文化底蕴 |
| **触点** | 报告页 → 一键分享 |
| **流程** | 查看报告 → 生成精美分享图 → 选择平台分享 |

**关键交互：**
1. 报告页底部固定"分享到朋友圈"按钮
2. 自动生成 2 张分享图（横版/竖版）
3. 分享后引导"添加客服领起名大全"

---

### 场景 4：起名决策 - 家族辈分参考

| 要素 | 内容 |
|-----|------|
| **场景名称** | 参考家族辈分给孩子起名 |
| **用户** | 孕晚期，需要确定宝宝名字 |
| **动机** | 遵循家族传统，兼顾文化内涵 |
| **触点** | 报告页 → 起名建议 |
| **流程** | 查看家族辈分表 → AI 生成推荐名字 → 收藏/分享 |

**关键交互：**
1. 报告中展示家族辈分传统
2. "AI 智能起名"按钮，基于姓氏、家族、辈分生成
3. 名字可收藏、对比、分享给家人

## 2. 关键用户流程图

```
用户旅程：
                                    
[发现] → [输入姓氏] → [免费预览] → [付费解锁] → [完整报告] → [起名建议] → [分享]
  ↓                    ↓            ↓            ↓            ↓            ↓
小红书/抖音         首次体验      价值感知      转化付费      深度服务      口碑传播
```

## 3. 页面结构

| 页面 | 功能 | 优先级 |
|-----|------|-------|
| 首页 | 姓氏输入 + 开始按钮 | P0 |
| 姓氏起源预览 | 免费展示起源故事 | P0 |
| 完整报告 | 付费解锁，完整内容 | P0 |
| 起名建议 | AI 推荐名字 | P1 |
| 分享页 | 精美分享图生成 | P1 |
| 我的收藏 | 收藏的名字、历史 | P2 |

---

## 4. 场景优先级

| 优先级 | 场景 | 理由 |
|-------|------|------|
| 1 | 姓氏溯源体验 | 核心价值，必须流畅 |
| 2 | 免费预览 → 付费转化 | 商业闭环关键 |
| 3 | 分享传播 | 获客来源 |
| 4 | 起名建议 | 增值服务 |
